Ideal Low-Pass Filter

Posted on Jun 9 2014   |   In Image processing   |  

%IDEAL LOW-PASS FILTER

%Part 1

f=imread(X);                          % reading an image X

%% f = rgb2gray(f);                    % if the image is gray instead of colour, rgb2gray can help convert 3 channels into 2 channels

P = 20;                             % P=input('Enter the cut-off frequency'); 

[M,N]=size(f);                         % Saving the the rows of X in M and columns in N

F=fft2(double(f));                     % Taking Fourier transform to the input image

%Part 2 % Finding the distance matrix D which is required to create the filter mask

u=0:(M-1);
v=0:(N-1);      
idx=find(u>M/2);   
u(idx)=u(idx)-M;
idy=find(v>N/2);
v(idy)=v(idy)-N;
[V,U]=meshgrid(v,u);
D=sqrt(U.^2+V.^2);

%Part 3

H=double(D<=P);                        % Comparing with the cut-off frequency to create filter mask 
G=H.*F;                               % Multiplying the Fourier transformed image with H
g=real(ifft2(double(G)));             % Inverse Fourier transform
imshow(f),figure,imshow(g,[ ]);       % Displaying input and output image

Concept of ILPF

It is a classical ideal low-pass filter. Before explaining these codes. I would like to review the ideal low-pass filter.

\[\begin{aligned}
H(u,v) =1, & \ D(u,v)\le{D_{0}}. \\
0, & \ D(u,v)>D_{0}.
\end{aligned} \]

From the above equations, the ideal low-pass filter will filter out the data which is larger then $D_{0}$.

And the general process of ILPF is

  1. shift the center := $ H(u,v) * (-1)^{x+y}$
  2. Use DFT := $F(u,v)$
  3. Apply the filter := $H(u,v)*F(u,v)$
  4. Compute the IDFT
  5. Get the real part
  6. Multiply := $(-1)^{x+y}$.

And for the above code, you can see that it skip the $(-1)^{x+y}$, which represents the shift part, it seems an error, but the I combine it into part 2.

u=0:(M-1);
v=0:(N-1);     
idx=find(u>M/2);   
u(idx)=u(idx)-M;
idy=find(v>N/2);
v(idy)=v(idy)-N;
[V,U]=meshgrid(v,u);

The first 6 lines is to find the matrix which represents this image. There is one thing to reminder reader that the centre of this image is not the centre of original image. The left upper corner of this image := [U V] is the center of original image, and the [U V]=meshgrid(v,u) is to build a image which is the original image.

From the picture, we can find the biggest part is the middle. And the left upper corner is zero. In part 3, the H=double(D<=P) to filter out the data which is larger than $D_{0}$.


The image is as same as our expectation. the left part is corner. which represents the centre of original image.
